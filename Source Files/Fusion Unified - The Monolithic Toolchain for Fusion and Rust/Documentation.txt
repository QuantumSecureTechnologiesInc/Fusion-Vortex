Fusion Unified v3.4The Monolithic Rust & Fusion Toolchain. Build, Audit, Test, and Run—all in one shared-memory process.1. Quick StartInstallationClone the repository and build the Fusion binary itself (bootstrap).cargo build --release
cp target/release/fusion_unified /usr/local/bin/fusion
The run CommandThis is the primary way to interact with Fusion. It performs the entire lifecycle in order.fusion run
What happens inside:Resolve: Calculates dependency graph.Audit: Checks selected versions against RustSec DB (Shift-Left).Compile: Builds crates in parallel.Test: Runs unit/integration tests.Execute: Launches your application and streams stdout/stderr to the Fusion dashboard.2. CLI ReferenceCommandDescriptionfusion runFull pipeline: Resolve -> Audit -> Build -> Test -> Execute.fusion buildStops after compilation. Useful for CI checking.fusion auditOnly runs the security scan.fusion watchDaemon mode. Keeps LSP server alive and rebuilds on file change.fusion agentAdvanced. Spawns a dedicated, single-purpose agent process.3. Architecture Upgrade (v3.1)Dashboard TUIFusion v3.1 introduces a TUI (Terminal User Interface) loop running on a separate Tokio thread.Mechanism: It reads the FusionState RwLock every 100ms.Rendering: It uses ANSI escape codes to redraw the status line at the bottom of the screen, while allowing application logs to scroll above it.Integrated Test RunnerThe BuildStatus enum now includes a Testing state.Fusion executes tests before the application starts.If tests fail, the run command aborts, preventing broken code from launching.Process InjectionWhen fusion run reaches the execution phase, it spawns the child process and pipes its stdout into the FusionState's stdout_buffer. The TUI thread picks this up and displays it, creating a seamless "Integrated Terminal" experience.4. Advanced Features: Autonomous AgentsFusion v3.4 introduces Dedicated Agents. These are specialized, isolated workers that can be deployed independently. To enforce strict separation of concerns, each agent is functionally locked to its domain and cannot perform tasks outside of it (e.g., an Audit Agent cannot compile code).1. The Sentinel (Audit Agent)Role: Dedicated Security & Compliance.Function: Monitors dependencies and source code for vulnerabilities. It operates on a read-only basis regarding the codebase but has write access to security logs.Capabilities:Periodic Scanning: Can be configured to wake up and scan on a cron schedule.Event-Driven: Triggers automatically on-file-create or on-lockfile-change.Isolation: Cannot trigger builds or run code.# Example: Spawn an audit agent that runs every hour AND on new file creation
fusion agent start --role audit --trigger "cron:0 * * * *" --watch "src/**/*"
2. The Constructor (Build Agent)Role: Pure Artifact Generation.Function: Resolves dependency graphs and compiles source code into binaries or libraries.Capabilities:Flux-Resolve Host: Manages the GPU-accelerated SAT solver.Artifact Caching: optimized for writing to the ./target directory.Isolation: Cannot execute the binaries it builds. It hands off artifacts to the Operator.# Example: dedicated build server mode
fusion agent start --role build --target release
3. The Operator (Run Agent)Role: Execution & Testing Environment.Function: Consumes artifacts from the Constructor to run tests or launch the final application.Capabilities:Sandboxing: Runs processes in isolated namespaces.Test Orchestration: manages unit tests, integration tests, and benchmarks.Isolation: Cannot compile code. If artifacts are missing, it returns a DependencyError rather than attempting a build.# Example: Run tests strictly using existing artifacts
fusion agent start --role run --mode test-only
4. The Observer (Intelligence Agent)Role: Analysis & Telemetry.Function: Powers the LSP (Language Server Protocol) and static analysis (fusion check).Capabilities:Zero-Copy Analysis: Reads memory directly from the Constructor's cache if available.Deep Lints: Performs semantic analysis without code generation.Isolation: Read-only access to the project. Cannot modify files or run binaries.# Example: Start the LSP agent for IDE integration
fusion agent start --role intelligence --port 9000
