Fusion Unified v3.1The Monolithic Rust Toolchain. Build, Audit, Test, and Run—all in one shared-memory process.1. Quick StartInstallationClone the repository and build the Fusion binary itself (bootstrap).cargo build --release
cp target/release/fusion_unified /usr/local/bin/fusion
The run CommandThis is the primary way to interact with Fusion. It performs the entire lifecycle in order.fusion run
What happens inside:Resolve: Calculates dependency graph.Audit: Checks selected versions against RustSec DB (Shift-Left).Compile: Builds crates in parallel.Test: Runs unit/integration tests.Execute: Launches your application and streams stdout/stderr to the Fusion dashboard.2. CLI ReferenceCommandDescriptionfusion runFull pipeline: Resolve -> Audit -> Build -> Test -> Execute.fusion buildStops after compilation. Useful for CI checking.fusion auditOnly runs the security scan.fusion watchDaemon mode. Keeps LSP server alive and rebuilds on file change.3. Architecture Upgrade (v3.1)Dashboard TUIFusion v3.1 introduces a TUI (Terminal User Interface) loop running on a separate Tokio thread.Mechanism: It reads the FusionState RwLock every 100ms.Rendering: It uses ANSI escape codes to redraw the status line at the bottom of the screen, while allowing application logs to scroll above it.Integrated Test RunnerThe BuildStatus enum now includes a Testing state.Fusion executes tests before the application starts.If tests fail, the run command aborts, preventing broken code from launching.Process InjectionWhen fusion run reaches the execution phase, it spawns the child process and pipes its stdout into the FusionState's stdout_buffer. The TUI thread picks this up and displays it, creating a seamless "Integrated Terminal" experience.
