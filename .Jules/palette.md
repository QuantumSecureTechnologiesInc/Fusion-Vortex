## 2024-07-04 - Converted Div Action Cards to Native Buttons
**Learning:** The dashboard used `div` elements for interactive "Run Tests", "Compile", and "Deliver" action cards, making them inaccessible to keyboard users and screen readers. When converting these to semantic `<button>` elements, we must also reset default button styles (like `text-align: center`, `width`, and `font-family`) in the CSS module to preserve the existing visual design while gaining native accessibility benefits.
**Action:** Always prefer native `<button>` elements for interactive UI components. When replacing existing `div` components, apply CSS reset properties (`text-align: left`, `display: block`, `width: 100%`, `color: inherit`, `font-family: inherit`) to prevent visual regressions while ensuring keyboard navigability and semantic meaning.
