# Fusion v2.0 Vortex Feature Index

Welcome to the feature documentation for the **Fusion v2.0 Vortex Programming Language**. This index tracks the unique, cutting-edge capabilities that distinguish Fusion from traditional systems languages.

Fusion is a native language: source files are `.fu` and projects are defined by `Fusion.toml`.

## 🚀 Core Pillars

### [Native Quantum Integration](./Quantum_Integration.md)
Fusion is the first language to treat qubits and quantum circuits as primitive types, enabling seamless hybrid quantum-classical computing.

### [Embedded AI Primitives](./AI_Primitives.md)
Deep learning is built into the standard library. Train and run models (LLMs, Neural Networks) without external Python dependencies.

### [Supernova Runtime](./Supernova_Runtime.md)
A tribrid execution engine that automatically dispatches code to the best available hardware: CPU, GPU, or QPU.

---

## 🛠️ Tooling & Experience

### [Advanced AI CLI](./Advanced_AI_CLI.md)
An offline-capable coding assistant that integrates a VS Code runtime, enabling extension usage directly from the terminal.

### [Visual Compiler](./Visual_Compiler.md)
Generate complete, compile-ready projects from natural language requirements. Supports Desktop, Web, and Native targets.

### [Fusion Forge](./Fusion_Forge.md)
A polyglot build system that handles Rust, C++, Python, and JS dependencies with a SAT-solver-based resolution engine.

---

## 🔒 Security & Connectivity

### [Post-Quantum Cryptography](./Post_Quantum_Cryptography.md)
Future-proof security with NIST-standardized algorithms (Kyber, Dilithium) built into the core crypto library.

### [Web & Networking](./Web_Networking.md)
Async-first HTTP/3 and gRPC server stack with native WebAssembly compilation support.
