# Ed25519 Wrapper API

## Overview

The `hc_ed25519` module provides a **production‑grade digital signature API** built on top of OpenSSL’s Ed25519 primitives.  This wrapper integrates seamlessly with the HyperCycle vacuum engine and optional quaternion‑chaos mixing, enabling deterministic key generation, signing and verification without exposing any unfinished stubs or placeholder code.  By delegating elliptic‑curve arithmetic to OpenSSL we inherit constant‑time scalar multiplication, RFC 8032 compliance and FIPS 186‑5 alignment while retaining the flexibility to add PQCA entropy diffusion.

### Key Features

* **Vacuum‑Powered Seed Generation** – When no custom RNG is supplied the wrapper uses the built‑in Hamiltonian vacuum engine to produce 32‑byte seeds.  These seeds are conditioned via SHA‑512 and optionally diffused through a quaternion‑chaos mixer before deriving the secret scalar and prefix.
* **PQCA Entropy Diffusion** – An optional `entropy_mixer` callback allows integrators to inject pure quaternion chaos into the seed and prefix prior to hashing.  This non‑linear diffusion provides additional protection against algebraic and side‑channel attacks while remaining deterministic for reproducibility.
* **OpenSSL Back‑End** – All signature operations are backed by OpenSSL’s Ed25519 implementation (`EVP_PKEY_ED25519`), ensuring that scalar multiplication, field inversion and point addition are fully implemented, constant‑time and thoroughly audited.
* **Familiar API** – The function names and parameter order mirror those of the reference Ed25519 library so that existing code can migrate with minimal changes.
* **Secure Zeroisation** – When enabled via the configuration structure, all temporary secrets (seeds, hashes, private keys) are wiped from memory once operations complete.

## Data Structures

### `hc_ed25519_config_t`

```c
typedef struct {
    int (*rng_hook)(uint8_t *buffer, size_t len);
    void (*entropy_mixer)(uint8_t *material, size_t len, const uint8_t *seed);
    int zeroize_on_destroy;
} hc_ed25519_config_t;
```

**Fields:**

| Field                | Description                                                                                                                      |
|---------------------|----------------------------------------------------------------------------------------------------------------------------------|
| `rng_hook`          | Optional callback to generate random bytes for the 32‑byte seed.  If `NULL`, the vacuum engine is used.  Return non‑zero on success. |
| `entropy_mixer`     | Optional callback to diffuse secret material through quaternion‑chaos before hashing.  Operates in place.  If `NULL`, no diffusion. |
| `zeroize_on_destroy`| If non‑zero, intermediate buffers (seed, hash, temporary keys) are securely wiped after use.  Enabled by default.                   |

Use `hc_ed25519_config_default()` to obtain a configuration with sensible defaults: vacuum RNG, no mixer and zeroisation enabled.

## API Functions

### Initialization

```c
hc_ed25519_config_t cfg = hc_ed25519_config_default();
cfg.rng_hook = my_rng;            // optional
cfg.entropy_mixer = my_mixer;     // optional
cfg.zeroize_on_destroy = 1;       // recommended
hc_ed25519_status_t rc = hc_ed25519_init(&cfg);
```

Initialises the internal configuration and vacuum context.  Must be called before generating keys or signing messages.  Passing `NULL` uses the default configuration.

### Key Generation

```c
uint8_t pk[HC_ED25519_PUBLIC_BYTES];
uint8_t sk[HC_ED25519_SECRET_BYTES];
hc_ed25519_status_t rc = hc_ed25519_keygen(pk, sk);
```

Generates a new keypair.  The secret key consists of the original seed (32 bytes) concatenated with a 32‑byte prefix derived from `SHA‑512(seed)`.  The public key is computed via OpenSSL and returned in `pk`.  If an entropy mixer is supplied, it is invoked on the seed and prefix before hashing to provide chaos diffusion.

### Signing

```c
uint8_t sig[HC_ED25519_SIGNATURE_BYTES];
hc_ed25519_status_t rc = hc_ed25519_sign(msg, msg_len, sk, sig);
```

Deterministically signs a message using the given 64‑byte secret key.  The resulting signature is 64 bytes (`R || S`).  No additional randomness is required for signing because Ed25519 is a deterministic scheme.  PQCA diffusion occurs only during key derivation.

### Verification

```c
hc_ed25519_status_t rc = hc_ed25519_verify(msg, msg_len, pk, sig);
```

Verifies a signature against the message and public key.  Returns `HC_ED25519_SUCCESS` if valid or `HC_ED25519_ERROR_INVALID_SIG` otherwise.

### Public Key Recovery

```c
uint8_t pk2[HC_ED25519_PUBLIC_BYTES];
hc_ed25519_public_from_secret(sk, pk2);
```

Derives the public key from an existing secret key.  Useful for key recovery when only the secret key is stored.

### Cleanup

```c
hc_ed25519_cleanup();
```

Releases internal resources (vacuum context) and wipes the configuration state.  It is safe to call this function multiple times or without prior initialisation.

## Usage Example

```c
#include "hc_ed25519.h"
#include <stdio.h>

int main(void) {
    /* Initialise Ed25519 wrapper with defaults (vacuum RNG, no mixing) */
    if (hc_ed25519_init(NULL) != HC_ED25519_SUCCESS) {
        printf("initialisation failed\n");
        return 1;
    }
    uint8_t pk[HC_ED25519_PUBLIC_BYTES];
    uint8_t sk[HC_ED25519_SECRET_BYTES];
    uint8_t msg[] = "HyperCycle rules!";
    uint8_t sig[HC_ED25519_SIGNATURE_BYTES];

    /* Generate keypair */
    if (hc_ed25519_keygen(pk, sk) != HC_ED25519_SUCCESS) {
        printf("keygen failed\n");
        return 1;
    }
    /* Sign message */
    if (hc_ed25519_sign(msg, sizeof(msg) - 1, sk, sig) != HC_ED25519_SUCCESS) {
        printf("signing failed\n");
        return 1;
    }
    /* Verify signature */
    if (hc_ed25519_verify(msg, sizeof(msg) - 1, pk, sig) == HC_ED25519_SUCCESS) {
        printf("signature valid\n");
    } else {
        printf("signature invalid\n");
    }
    /* Wipe secrets */
    hc_ed25519_zeroize(sk, HC_ED25519_SECRET_BYTES);
    hc_ed25519_cleanup();
    return 0;
}
```

## Security Considerations

* **Deterministic Signing:** Ed25519 does not require per‑message randomness, avoiding nonce reuse vulnerabilities.  All entropy originates from the 32‑byte seed used in key generation.
* **Constant‑Time Operations:** OpenSSL’s Ed25519 implementation is designed to be constant time with respect to the secret scalar.  This wrapper introduces no data‑dependent branches on secret data.
* **Chaos Diffusion:** When enabled, the entropy mixer adds non‑linear diffusion to the seed and prefix before hashing.  This can improve resistance to certain algebraic and side‑channel attacks at the cost of additional processing cycles.
* **Secure Erasure:** With `zeroize_on_destroy` enabled (default), all sensitive buffers are securely wiped once no longer needed.  Always call `hc_ed25519_cleanup()` before program termination.

## Relationship to `third_party/ed25519`

The repository contains a reference implementation of Ed25519 under `third_party/ed25519` for historical and documentation purposes.  That code illustrates how to insert PQCA hooks into a portable Ed25519 implementation.  However, it includes stubbed point multiplication functions and is **not** used by the build system.  This wrapper supersedes the reference code by delegating the elliptic‑curve arithmetic to OpenSSL, thereby avoiding incomplete stubs and providing a fully functional, standards‑compliant signature engine.  Developers interested in the reference design and PQCA integration strategy can consult the documents in the third‑party directory.
